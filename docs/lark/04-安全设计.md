# AA Wallet 安全设计

## 1. 多层验证机制

```
┌─────────────────────────────────────────────────────────────────┐
│                    executeBatch 输入                             │
│                                                                 │
│   UserOperation  用户操作数据                                    │
│   signature      用户签名                                        │
│   caller         调用者(Relayer地址)                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  第一层: Relayer验证                             │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  检查 Relayer 是否在白名单                   │              │
│  │                                             │              │
│  │  if (!_relayers[msg.sender]) {              │              │
│  │      revert NotRelayer();                   │              │
│  │  }                                          │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│   失败 → ❌ NotRelayer 错误                                     │
│   成功 → 继续                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  第二层: 签名验证                                 │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  步骤①: 构造hash                            │              │
│  │  hash = keccak256(abi.encode(userOp))       │              │
│  │                                             │              │
│  │  步骤②: 添加EIP-191前缀                     │              │
│  │  ethSignedHash = hash.toEthSignedMessage()  │              │
│  │                                             │              │
│  │  步骤③: ECDSA恢复signer                     │              │
│  │  signer = ethSignedHash.recover(signature)  │              │
│  │                                             │              │
│  │  步骤④: 验证signer                          │              │
│  │  if (signer != userOp.user) {               │              │
│  │      revert InvalidSignature();             │              │
│  │  }                                          │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│   失败 → ❌ InvalidSignature 错误                               │
│   成功 → 继续                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  第三层: 执行验证                                 │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  遍历执行所有calls                          │              │
│  │                                             │              │
│  │  for (i = 0; i < calls.length; i++) {       │              │
│  │      (success, ) = calls[i].to.call(data);  │              │
│  │      if (!success) revert CallFailed();     │              │
│  │  }                                          │              │
│  │                                             │              │
│  │  记录gas消耗                                 │              │
│  │  gasUsed = gasBefore - gasleft();           │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│   失败 → ❌ CallFailed 错误                                     │
│   成功 → 继续                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  第四层: 补偿验证                                 │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  计算补偿金额                                │              │
│  │                                             │              │
│  │  执行USDT转账                                │              │
│  │  if (!usdt.transferFrom(user, relayer, ...))│              │
│  │      revert TransferFailed();               │              │
│  │                                             │              │
│  │  执行手续费转账                              │              │
│  │  if (fee > 0 && !usdt.transferFrom(...))    │              │
│  │      revert TransferFailed();               │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│   失败 → ❌ TransferFailed 错误                                 │
│   成功 → ✓ 交易完成                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 权限控制矩阵

```
┌─────────────────────────────────────────────────────────────────┐
│                     合约方法                                     │
│                                                                 │
│   executeBatch()     ←── 仅 Relayer 白名单                      │
│   addRelayer()       ←── 仅 Owner                               │
│   removeRelayer()    ←── 仅 Owner                               │
│   setFeeRate()       ←── 仅 Owner                               │
│   setOracle()        ←── 仅 Owner                               │
│   setFeeRecipient()  ←── 仅 Owner                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     权限角色                                     │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  Relayer白名单                              │              │
│  │                                             │              │
│  │  权限: executeBatch()                       │              │
│  │                                             │              │
│  │  添加: Owner调用 addRelayer(address)        │              │
│  │  移除: Owner调用 removeRelayer(address)     │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│   ┌─────────────────────────────────────────────┐              │
│  │  合约Owner                                  │              │
│  │                                             │              │
│  │  权限: 所有管理方法                          │              │
│  │                                             │              │
│  │  设置: 初始化时指定                          │              │
│  │  转移: Ownable的transferOwnership()         │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 3. 权限详情表

| 操作 | 权限要求 | 说明 |
|------|----------|------|
| executeBatch | Relayer白名单 | 只有白名单Relayer可调用 |
| addRelayer | 合约Owner | 添加Relayer到白名单 |
| removeRelayer | 合约Owner | 从白名单移除Relayer |
| setFeeRate | 合约Owner | 设置手续费率 |
| setOracle | 合约Owner | 更新Oracle地址 |
| setFeeRecipient | 合约Owner | 设置手续费接收地址 |

## 4. 安全防护措施

```
┌─────────────────────────────────────────────────────────────────┐
│                     安全防护                                     │
│                                                                 │
│  签名验证                                                       │
│  ┌─────────────────────────────────────────────┐              │
│  │  • EIP-7701 authorization签名               │              │
│  │  • EIP-1271 智能合约签名验证                 │              │
│  │  • ECDSA 恢复验证                            │              │
│  │  • chainId 包含在hash中                     │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│  重放攻击防护                                                   │
│  ┌─────────────────────────────────────────────┐              │
│  │  • chainId 包含在签名hash                   │              │
│  │  • nonce 防止同一操作重复                   │              │
│  │  • 签名唯一性验证                            │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│  权限控制                                                       │
│  ┌─────────────────────────────────────────────┐              │
│  │  • Relayer白名单                            │              │
│  │  • Owner管理权限                             │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
│  重入攻击防护                                                   │
│  ┌─────────────────────────────────────────────┐              │
│  │  • ReentrancyGuard                          │              │
│  │  • nonReentrant 修饰符                       │              │
│  └─────────────────────────────────────────────┘              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 签名验证流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     签名构造                                     │
│                                                                 │
│  步骤①: 构造UserOperation                                      │
│  user = 用户地址                                                │
│  calls = [{to, data}, ...]                                     │
│  nonce = 当前nonce                                             │
│                                                                 │
│  步骤②: 计算hash                                               │
│  hash = keccak256(abi.encode(userOp))                          │
│                                                                 │
│  步骤③: 添加EIP-191前缀                                        │
│  ethSignedHash = keccak256(                                    │
│      "\x19Ethereum Signed Message:\n32" + hash                 │
│  )                                                              │
│                                                                 │
│  步骤④: 用户钱包签名                                           │
│  signature = eth_sign(ethSignedHash)                           │
│  → {r, s, v} 共65字节                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     签名验证                                     │
│                                                                 │
│  步骤①: ECDSA恢复                                              │
│  signer = ecrecover(ethSignedHash, v, r, s)                    │
│                                                                 │
│  步骤②: 验证signer                                             │
│  require(signer == userOp.user, "InvalidSignature")            │
│                                                                 │
│  ✓ 验证通过 → 执行操作                                         │
│  ✗ 验证失败 → revert                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
