# AA Wallet 技术设计文档

## 1. 项目概述

AA Wallet 是基于 EIP-7702 的账户抽象钱包，用户无需持有 BNB 即可进行 USDT 转账，Gas 费用由 Relayer 代付并以 USDT 形式从用户账户扣除。

**核心特性：**

| 特性 | 描述 |
|------|------|
| Gasless 交易 | 用户无需持有 BNB 即可发起交易 |
| EIP-7702 授权 | EOA 账户临时转换为智能合约账户 |
| USDT 支付 Gas | 用 USDT 支付交易费用 |
| 批量操作 | 单次签名执行多个操作（approve + transfer） |
| Relayer 网络 | 分布式 Relayer 池代付 Gas |

## 3. EIP-7702 代理合约机制

```
┌─────────────────────────────────────────────────────────────────┐
│                  EIP-7702 核心原理                                │
│                                                                 │
│   传统模式:                                                      │
│   ┌─────────────────────────────────────────────┐              │
│   │                                             │              │
│   │   用户 EOA (普通账户)                        │              │
│   │   address: 0x1234...                        │              │
│   │   code: 空 (无合约代码)                      │              │
│   │                                             │              │
│   │   限制:                                      │              │
│   │   • 只能发起交易，不能执行合约逻辑           │              │
│   │   • 必须持有 BNB 支付 Gas                    │              │
│   │   • 无法批量执行多个操作                     │              │
│   │                                             │              │
│   └─────────────────────────────────────────────┘              │
│                                                                 │
│   EIP-7702 模式:                                                 │
│   ┌─────────────────────────────────────────────┐              │
│   │                                             │              │
│   │   用户 EOA + Simple7702Account 代码         │              │
│   │   address: 0x1234... (地址不变)             │              │
│   │   code: Simple7702Account (临时绑定)        │              │
│   │                                             │              │
│   │   通过 EIP-7702 setCode 操作:               │              │
│   │   用户签名 authorization                    │              │
│   │   Relayer 提交 setCode 交易                 │              │
│   │   链上将 Simple7702Account 代码             │              │
│   │   绑定到用户 EOA 地址                       │              │
│   │                                             │              │
│   │   效果:                                      │              │
│   │   • 用户 EOA 现在拥有合约代码               │              │
│   │   • 可以执行 executeBatch 等合约方法       │              │
│   │   • 可以进行批量操作                        │              │
│   │   • Paymaster 可代付 Gas                    │              │
│   │                                             │              │
│   └─────────────────────────────────────────────┘              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Simple7702Account 代理合约作用

```
┌─────────────────────────────────────────────────────────────────┐
│              Simple7702Account 作为 EIP-7702 代理合约             │
│                                                                 │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   Simple7702Account 合约                               │    │
│   │   地址: 0x9e4e06F875464EEB3aE0AA7993243f910f119Bee     │    │
│   │                                                        │    │
│   │   作为"代码模板":                                      │    │
│   │   • 不存储用户状态                                     │    │
│   │   • 提供合约逻辑给绑定的 EOA                           │    │
│   │                                                        │    │
│   │   核心功能:                                            │    │
│   │   ┌─────────────────────────────────────────────┐     │    │
│   │   │ executeBatch(calls[])                       │     │    │
│   │   │                                             │     │    │
│   │   │ • 执行批量调用 (最多5个)                     │     │    │
│   │   │ • 每个 call 包含 {to, data}                 │     │    │
│   │   │ • 示例: [approve, transfer]                 │     │    │
│   │   │                                             │     │    │
│   │   │ 限制: 仅 Paymaster 可调用                   │     │    │
│   │   │                                             │     │    │
│   │   └─────────────────────────────────────────────┘     │    │
│   │                                                        │    │
│   │   ┌─────────────────────────────────────────────┐     │    │
│   │   │ isValidSignature(hash, signature)           │     │    │
│   │   │                                             │     │    │
│   │   │ • EIP-1271 标准签名验证                      │     │    │
│   │   │ • 验证用户是否授权了该操作                   │     │    │
│   │   │                                             │     │    │
│   │   │ 验证流程:                                    │     │    │
│   │   │ 1. ECDSA 恢复 signer                        │     │    │
│   │   │ 2. 比较 signer == EOA owner                 │     │    │
│   │   │ 3. 返回 0x1626ba7e (有效) 或 0xffffffff      │     │    │
│   │   │                                             │     │    │
│   │   └─────────────────────────────────────────────┘     │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
│   用户 EOA 绑定后:                                              │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   用户 EOA 地址: 0x1234...                             │    │
│   │                                                        │    │
│   │   执行交易时:                                          │    │
│   │   • msg.sender = 用户 EOA 地址                        │    │
│   │   • 但执行的是 Simple7702Account 的代码逻辑           │    │
│   │   • 用户签名验证使用 EOA 的 owner                     │    │
│   │                                                        │    │
│   │   Paymaster 调用流程:                                  │    │
│   │   1. Paymaster.executeBatch(userOp, sig)              │    │
│   │   2. Paymaster 调用 user.executeBatch(calls)          │    │
│   │      (user 是已绑定代码的 EOA 地址)                    │    │
│   │   3. EOA 执行 Simple7702Account 的 executeBatch       │    │
│   │   4. 执行 calls[] 中的操作                             │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 两个 executeBatch 的区别

```
┌─────────────────────────────────────────────────────────────────┐
│                executeBatch 方法职责划分                          │
│                                                                 │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   ① Paymaster.executeBatch(userOp, signature)         │    │
│   │      入口方法 - Relayer 调用                           │    │
│   │                                                        │    │
│   │      参数:                                             │    │
│   │      • userOp: UserOperation 结构                      │    │
│   │        - user: 用户EOA地址                             │    │
│   │        - calls: 操作列表                               │    │
│   │        - nonce: 防重放                                 │    │
│   │      • signature: 用户签名                             │    │
│   │                                                        │    │
│   │      职责:                                             │    │
│   │      ┌─────────────────────────────────────────┐      │    │
│   │      │ 1. 验证 Relayer 白名单                   │      │    │
│   │      │ 2. 验证用户签名 (ECDSA恢复)              │      │    │
│   │      │ 3. 调用 user.executeBatch(calls)        │      │    │
│   │      │    ↓                                      │      │    │
│   │      │    (触发 Simple7702Account 代码执行)     │      │    │
│   │      │ 4. 计算 Gas 补偿                         │      │    │
│   │      │ 5. 执行 USDT 转账补偿                    │      │    │
│   │      └─────────────────────────────────────────┘      │    │
│   │                                                        │    │
│   │      不执行实际操作，只做验证和补偿                     │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                              │                                  │
│                              │ 调用 user.executeBatch(calls)   │
│                              │ (user 是已绑定代码的 EOA)        │
│                              ↓                                  │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   ② Simple7702Account.executeBatch(calls[])           │    │
│   │      执行方法 - 实际执行操作                           │    │
│   │                                                        │    │
│   │      参数:                                             │    │
│   │      • calls: Call[] 操作列表                          │    │
│   │        - to: 目标合约地址                              │    │
│   │        - data: 调用数据                                │    │
│   │                                                        │    │
│   │      调用者限制:                                        │    │
│   │      ┌─────────────────────────────────────────┐      │    │
│   │      │ modifier onlyPaymaster() {              │      │    │
│   │      │   require(msg.sender == paymaster);     │      │    │
│   │      │ }                                       │      │    │
│   │      └─────────────────────────────────────────┘      │    │
│   │                                                        │    │
│   │      职责:                                             │    │
│   │      ┌─────────────────────────────────────────┐      │    │
│   │      │ 遍历执行 calls[]                         │      │    │
│   │      │                                         │      │    │
│   │      │ for (i = 0; i < calls.length; i++) {    │      │    │
│   │      │   (success,) = calls[i].to.call(data);  │      │    │
│   │      │   require(success);                     │      │    │
│   │      │ }                                       │      │    │
│   │      └─────────────────────────────────────────┘      │    │
│   │                                                        │    │
│   │      只执行操作，不做验证和补偿                        │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
│   总结:                                                         │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   Paymaster.executeBatch    →  验证层 + 补偿层         │    │
│   │   Simple7702Account.executeBatch →  执行层             │    │
│   │                                                        │    │
│   │   调用链:                                              │    │
│   │   Relayer → Paymaster.executeBatch                    │    │
│   │           → user.executeBatch (7702代理合约代码)       │    │
│   │           → 执行实际操作                               │    │
│   │           → 返回 Paymaster                             │    │
│   │           → 计算补偿                                   │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 完整调用流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     合约交互流程                                  │
│                                                                 │
│   Relayer                                                        │
│      │                                                           │
│      │ 调用 executeBatch(userOp, signature)                      │
│      ↓                                                           │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                  USDTPaymaster                          │    │
│   │                                                        │    │
│   │   步骤①: 验证 Relayer 白名单                            │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ if (!_relayers[msg.sender]) │               │    │
│   │          │     revert NotRelayer()     │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                                                        │    │
│   │   步骤②: 验证用户签名                                   │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ signer = ECDSA.recover()    │               │    │
│   │          │ require(signer == user)     │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                                                        │    │
│   │   步骤③: 执行用户操作                                   │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ 调用 Simple7702Account       │               │    │
│   │          │   .executeBatch(calls)      │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                    ↓                                   │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ 执行 calls[] 内容           │               │    │
│   │          │ • approve(Paymaster, amt)   │               │    │
│   │          │ • transfer(target, amt)     │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                                                        │    │
│   │   步骤④: 计算 Gas 补偿                                  │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ gasUsed = gasleft差值       │               │    │
│   │          │ bnbCost = gasUsed × gasPrice│               │    │
│   │          └─────────────────────────────┘               │    │
│   │                    ↓                                   │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ 调用 PriceOracle             │               │    │
│   │          │   .getBNBPriceInUSDT()      │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                    ↓                                   │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ compensation = bnbCost      │               │    │
│   │          │   × bnbPrice ÷ 1e18         │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                                                        │    │
│   │   步骤⑤: 执行 USDT 补偿转账                             │    │
│   │          ┌─────────────────────────────┐               │    │
│   │          │ USDT.transferFrom(          │               │    │
│   │          │   user → Relayer            │               │    │
│   │          │ )                           │               │    │
│   │          └─────────────────────────────┘               │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

│   │                                                        │    │
│   │   ② 选择空闲 Relayer                                   │    │

## 7. Relayer池设计

```
┌─────────────────────────────────────────────────────────────────┐
│                      Relayer 池架构                              │
│                                                                 │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                   Pool 结构                             │    │
│   │                                                        │    │
│   │   relayers[]: Relayer列表                              │    │
│   │   mu: sync.RWMutex (读写锁)                            │    │
│   │                                                        │    │
│   │   方法:                                               │    │
│   │   • SelectIdle()    选择空闲Relayer                    │    │
│   │   • MarkPending()   标记开始处理                       │    │
│   │   • MarkComplete()  标记处理完成                       │    │
│   │   • GetAll()        获取所有状态                       │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                   Relayer 结构                          │    │
│   │                                                        │    │
│   │   address: Relayer 地址                                │    │
│   │   privateKey: Relayer 私钥                             │    │
│   │   pendingTx: 待处理交易数                              │    │
│   │   lastUsed: 上次使用时间                               │    │
│   │                                                        │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
│   选择策略: 负载均衡                                            │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                                                        │    │
│   │   规则: 选择 pending 最少的 Relayer                    │    │
│   │                                                        │    │
│   │   示例:                                                │    │
│   │   ┌─────────────────────────────────────┐             │    │

│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
